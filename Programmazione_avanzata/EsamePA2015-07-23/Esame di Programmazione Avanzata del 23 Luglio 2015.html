Exercise 1: Word of Mouth.

In computer science, edit distance is a way of quantifying how dissimilar two strings (e.g., words) are to one another by counting the minimum number of operations required to transform one string into the other. This is also the basis of a well known car game (a car game is one of those games that children play to spend time during a long travel in a car) that consists to a reach word from another by changing only one letter at each turn. Basically, one player decides a couple of words respectively the starting and ending word. All the players are sorted in some way and at each turn a player says a new word that differs from the previous one (the starting one if the player is the first one) of only one character. Wins who says the ending word.

The exercise consists of writing a function chain that calculates and returns all the possible paths that bring from the starting word to the ending one. The rules for passing from one word to anther are those above explained plus no word can be repeated in the path and all the used words should belong to this dictionary. The chain function will return the result as a string where each line contains a different path and all the path are lexicographically sorted. If no path is possible the result will be the empty string.

Note. The only acceptable solutions are those that uses recursion any other approach will be rejected.

from editd import *

if __name__ == '__main__':
  print("### witness → fatness")
  print(chain("witness", "fatness"))
  print("### warning → earring")
  print(chain("warning", "earring"))
  print("### sailing → writing")
  print(chain("sailing", "writing"))

Exercise 2: Up and Down over a File.

In python, files are iterable objects but it is impossible to push back in the stream what has already been read.

This is exercise is about reimplementing an iterator--called UpDownFile--over files that permits to get a word at each call of the next operator. A word is any piece of text without any, white space, punctuation symbol--including brackets, squared and curly parenthesis--and end of line/file. Moreover it supports a ungetw method that permits to push back a word at a time in the file stream. That is, a call to next after a call to ungetw will extract the last extracted word.

Note, any solution that do not implement an interator will be considered wrong; any solution that do not respect the given--by the example below--interface will be considered wrong; any solution that uses shortcuts from any standard library will be considered wrong: the only admitted methods on files are open and read.

The following is a possible main followed by the desired execution result. The text file used in the main can be downloaded from here.

from iter import *

if __name__ == "__main__":
  fiter = UpDownFile('wikipedia-excerpt.txt')
  iter(fiter)
  print('### Let\'s go up and down for a while') 
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  fiter.ungetw()
  fiter.ungetw()
  fiter.ungetw()
  print(next(fiter))
  print(next(fiter))
  fiter.ungetw()
  fiter.ungetw()
  print(next(fiter))
  fiter.ungetw()
  fiter.ungetw()
  fiter.ungetw()
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  fiter.ungetw()
  print('### Let\'s finish the iteration') 
  try:
    while True:
      print(next(fiter))
  except StopIteration: pass
  print('### Let\'s restart the iteration') 
  iter(fiter)
  print(next(fiter))
  print(next(fiter))
  print(next(fiter))
  fiter.ungetw()
  print(next(fiter))
